# Task Domain Concepts

---
## Task

### 정의
이 문서에서 task 는 atomic 한 작업의 단위이다.
더 이상 의미 있게 쪼개기 어려운 최소 실행 단위이며, 하나의 명확한 목표와 범위를 가진다.

### 역할
- Agent가 다른 Agent와 충돌 없이 작업할 수 있게 함
- 큰 작업을 실제 실행 가능한 최소 단위로 분해함
- 작업 범위와 책임을 명확하게 고정함
- 병렬 수행이 가능하도록 경계를 분리함
- 진행 상태와 완료 여부를 명확하게 판단할 수 있게 함
- milestone 의 계획을 실제 실행으로 전환함

### 외우면 좋은 규칙
- task 는 atomic 해야 한다.
- task 는 한 번에 하나의 명확한 목적만 가져야 한다.
- task 는 다른 task 와 책임이 겹치지 않아야 한다.
- task 는 독립적으로 수행 가능해야 한다.
- task 는 완료 조건이 분명해야 한다.
- task 는 milestone 으로부터 분배받아 실행된다.
- task 는 실행 단위이지, 큰 기획 단위가 아니다.

### 한 줄 암기법
task 는 "충돌 없이 분리되어 독립적으로 수행되는 최소 실행 단위"다.

## Summary

### 정의
이 문서에서 summary 는 작업을 마친 AI agent의 context의 압축 요약이다.

### 역할
- 완료한 작업의 핵심 결과 정리
- 현재 상태와 결정 사항 보존
- 남은 이슈와 다음 시작점 전달
- 긴 문맥 없이도 재진입 가능하게 만들기

### 규칙
- 길게 쓰지 않는다.
- 핵심만 남긴다.
- 결정된 것과 미결정을 분리한다.
- 다음 에이전트가 바로 이어받을 수 있게 쓴다.
- 세부 로그가 아니라 재시작 가능한 맥락을 남긴다.

### 한 줄 암기법
summary 는 "끝난 작업의 핵심만 압축해 다음 실행으로 넘기는 문맥"이다.

## Frontier

### 정의
현재 이 문서에서 frontier 는 1촌 부모 task 또는 milestone 으로부터 전달받는 현재 작업의 명령 문서를 의미한다.
이는 지금 시점에서 무엇을 수행해야 하는지, 어디까지 진행해야 하는지, 그리고 어떤 방향으로 이어가야 하는지를 분명하게 전달하기 위한 문서다.

### 역할
- 현재 작업의 직접적인 실행 방향을 지정한다.
- agent 가 지금 해야 할 일을 명확하게 이해할 수 있게 한다.
- 과거를 요약하는 것이 아니라, 곧바로 이어서 수행할 작업의 경계를 정한다.
- 부모 task 또는 milestone 의 의도를 현재 실행 단위로 전달한다.

### 규칙
- frontier 는 atomic 한 task 단위로 다뤄진다.
- frontier 는 1촌 부모 관계에서만 수직 하향으로 전달된다.
- frontier 는 현재 작업에 대한 명령서다.
- frontier 는 과거 기록이 아니라 다음 실행의 기준이다.
- frontier 는 지금 해야 할 일과 작업 범위를 분명하게 담아야 한다.
- agent 는 전달받은 frontier 를 기준으로 곧바로 작업을 이어간다.

### 한 줄 암기법
frontier 는 "부모가 자식에게 보내는 현재 작업의 직접 명령서"다.
